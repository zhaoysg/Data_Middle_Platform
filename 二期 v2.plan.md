## Step 0: Context Synthesis

### 0A. Premise Challenge

v2 确认的决策：

1. **审批流去掉** — 单人模式不需要审批；`sec_access_application`、`sec_access_audit`、`sec_mask_whitelist`、`sec_new_column_alert` 等审批相关表和逻辑全部删除
2. **以 ruoyi 为主重写** — 不桥接 bagdata 引擎，在 ruoyi `DqcExecutionServiceImpl` 基础上重写 DQC 执行逻辑
3. **前端全在 ruoyi-vben-ui 重写** — 不复制 bagdata 的 Vue2 代码，在 ruoyi 的 Vue3 技术栈上重新实现
4. **三块并行 Track** — 三个模块（DQC / 脱敏 / 数据探查）互不依赖地并行开发，只通过明确定义的接口交互

**Eureka v2**: 桥接策略的真正问题不是能不能桥，而是桥接后谁维护。bagdata 引擎是一个封闭系统，桥接后 ruoyi 团队无法独立演进。以 ruoyi 为主重写，虽然多花 1 周，但架构统一，长期可维护性更高。

## 三块并行 Track 设计（核心结构）

### Track 划分原则

三个 Track 互不直接依赖，可以并行开发，通过明确定义的 Service 接口交互：

```
公共基础（最先完成，0.5 天）：
  DatasourceHelper 工具类（复用 sys_datasource 的解密+建连逻辑）
  DqcQualityScore entity + service（三个 Track 共享写入）

Track A: DQC 数据质量检查
  依赖：无（独立模块）
  输出：DqcRuleTemplate CRUD + DqcRuleDef CRUD + DqcPlan CRUD + 执行引擎

Track B: 脱敏体系
  依赖：无（独立模块）
  输出：敏感识别规则 CRUD + 脱敏模板 CRUD + 脱敏策略 CRUD + MaskQuery API

Track C: 数据探查
  依赖：无（独立模块）
  输出：DprofileTask CRUD + 表/列统计分析 + 快照管理
```

### 公共基础（最先完成，0.5 天）

```java
DatasourceHelper
  getDecryptedJdbcTemplate(dsId): JdbcTemplate
  getAdapter(dsId): DataSourceAdapter

DqcQualityScore (entity + mapper + service)
  存储三个 Track 的评分结果（read 主要来自 DQC，write 三个 Track 都写）
```

## 前端设计规范（Design Review v1 新增）

### 设计原则

基于 `ruoyi-vben-ui` 技术栈（Vue3 + AntDesign Vue + VxeGrid + VbenDrawer），统一以下设计规范：

- **组件库**: AntDesign Vue 5.x，全部使用官方组件
- **表格**: `useVbenVxeGrid` + VxeGrid（参考 `catalog/index.vue`）
- **抽屉**: `useVbenDrawer`（参考 `table-drawer.vue`）
- **向导**: `useVbenModal` + 分步骤状态机（参考 bagdata `rule-def/index.vue` 的 `<a-steps>` 模式）
- **无独立详情页** — 所有详情用抽屉承载
- **向导内嵌在抽屉内** — 新建/编辑时弹出向导抽屉，而不是独立路由页面
- **响应式**: 统一 `grid-cols-1 md:grid-cols-2 lg:grid-cols-4` 表单布局

### 向导设计规范

#### 向导抽屉模板（通用结构）

所有向导统一使用以下模式：

```vue
<script setup lang="ts">
// 向导步骤状态
const currentStep = ref(0);
const totalSteps = ref(4);
const formData = ref({});
const steps = [
  { title: '基本信息', description: '填写名称和描述' },
  { title: '配置规则', description: '设置检测规则' },
  { title: '预览保存', description: '确认并保存' },
];

function handleNext() {
  if (currentStep.value < totalSteps.value - 1) {
    currentStep.value++;
  } else {
    handleSubmit();
  }
}

function handlePrev() {
  if (currentStep.value > 0) currentStep.value--;
}
</script>

<template>
  <BasicDrawer :title="`${isEdit ? '编辑' : '新建'}向导`" class="w-[720px]">
    <!-- 步骤条 -->
    <a-steps :current="currentStep" size="small" class="mb-6" />

    <!-- 步骤内容区 -->
    <div class="step-content">
      <template v-if="currentStep === 0">
        <!-- Step 1: 基本信息 -->
      </template>
      <template v-if="currentStep === 1">
        <!-- Step 2: 配置 -->
      </template>
      <template v-if="currentStep === 2">
        <!-- Step 3: 预览 -->
      </template>
    </div>

    <!-- 底部按钮 -->
    <div class="flex justify-end gap-3 mt-6 border-t pt-4">
      <a-button v-if="currentStep > 0" @click="handlePrev">上一步</a-button>
      <a-button v-if="currentStep < totalSteps - 1" type="primary" @click="handleNext">下一步</a-button>
      <a-button v-if="currentStep === totalSteps - 1" type="primary" :loading="submitting" @click="handleSubmit">保存</a-button>
    </div>
  </BasicDrawer>
</template>
```

### 功能交互规范（Design Review v2 新增 — 全部按钮状态明确）

#### G0 — 通用原则（所有页面强制遵守）

```
所有交互遵循以下铁律：
1. 按钮禁用时必须有明确原因（hover 显示 tooltip 说明）
2. loading 状态必须同时：loading=true + 按钮文字不变 + 按钮不可点
3. 操作完成后必须重置 loading 状态（无论成功或失败）
4. 所有 API 调用必须 try-catch，catch 分支也要关闭 loading
5. 删除类危险操作：按钮变红 + 点击后弹确认框 + 确认框内再次点击才执行
```

- `<a-table>` 分页提供"跳转到第 N 页"输入框（AntDesign 内置）
